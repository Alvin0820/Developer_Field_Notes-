Developer Field Notes

Git vs GitHub

Git is a version control system. GitHub is a cloud platform for hosting Git repositories.

Daily Loop

  1. Edit
  2. Stage
  3. Commit
  4. Push

Useful Git Commands

  • git status - check repository status
  • git add - stage files
  • git commit - create a snapshot
  • git push - upload commits
  • git pull - download changes
  • git clone - copy a repository
  • git branch - view branches
  • git checkout - switch branches

Things I Should Not Commit

  • .env files
  • API secrets
  • node_modules
  • .DS_Store

Command Reference

  • git statusShows the current state of the repository.
  • git addStages files for commit.
  • git commitCreates a snapshot of staged changes.
  • git push - Uploads commits to GitHub.
  • git pull - Downloads changes from GitHub.
  • git branch - Lists or creates branches.
  • git checkout - Switches branches.

Things I should not Commit

  • .env files
  • API keys
  • Secrets
  • node_modules
  • .DS_Store
